## Task 1 — Embeddings artifact (skippable if no OPENAI_API_KEY at hand)

Create **`scripts/ingest/embed-products.mjs`** (zero deps, Node 24, mirror the style of `scripts/ingest/build-products.mjs`):

1. Read `apps/web/src/data/products.json`.
2. For each product build the embed text: `` `${name}. ${tags.join(", ")}. ${description}` ``.
3. Call OpenAI `POST /v1/embeddings` with `model: "text-embedding-3-small"`, `dimensions: 256`, batching 100 inputs per request (5 requests total). Read the key from `process.env.OPENAI_API_KEY`; abort with a clear message if unset.
4. Round values to 5 decimals. Write `apps/web/src/data/embeddings.json` as `{ [productId]: number[256] }` (~1.3 MB).
5. Idempotent: if the output file already has all 500 ids, exit early with "up to date".

**Check:** `node scripts/ingest/embed-products.mjs` prints 500 embedded; the JSON has 500 keys, each a 256-length array. If you have no API key, SKIP this task entirely — Task 2's fallback covers it and nothing else depends on this file existing.

## Task 2 — Matching engine: `apps/web/src/lib/match.ts` (+ test)

Pure functions, no React. Import `flowers` from `../data/products`. Import embeddings with a graceful miss:

```ts
let vectors: Record<string, number[]> | null = null;
try { vectors = (await import("../data/embeddings.json")).default; } catch { vectors = null; }
```
(If top-level await fights the test setup, use a lazy `getVectors()` with the same try/catch — behavior over form.)

**Exports:**

```ts
export type GiftFilters = { occasion?: string; giftType?: string; budget?: string };
export function topMatches(filters: GiftFilters, excludeIds: Set<string>, count = 10): Flower[];
export function similarTo(likedIds: string[], excludeIds: Set<string>, count = 6): Flower[];
```

**Filter vocabularies (exact strings — the UI in Task 3 must use the same):**

```ts
export const GIFT_TYPES: Record<string, string[]> = {
  "Flowers & plants": ["Flowers", "Dried flowers", "Orchid", "Plant"],
  "Sweet treats": ["Cake", "Chocolate"],
  "Keepsakes": ["Mug", "Teddy bear", "Candle", "Balloon", "Money bouquet"],
  "Drinks & baskets": ["Wine & spirits", "Gift basket"],
  // "Surprise me" = no type filter
};
export const BUDGETS: Record<string, [number, number]> = {
  "Under ₱1,000": [0, 1000], "₱1,000–₱2,500": [1000, 2500],
  "₱2,500–₱5,000": [2500, 5000], "₱5,000+": [5000, Infinity],
};
export const OCCASIONS = ["Birthday", "Anniversary", "Congratulations", "Thank you", "Apology", "Sympathy"];
```

**`topMatches` logic (deterministic — no RNG):**
1. Hard filters: category (`flower.tags[0]`) must be in the chosen gift-type group; `pricing.amount` within budget range; not in `excludeIds`.
2. **Sympathy gate (non-negotiable):** if `filters.occasion !== "Sympathy"`, exclude any product whose category is `Sympathy` OR whose `occasions` include `"Sympathy"`. If occasion IS Sympathy, allow ONLY categories `["Sympathy", "Flowers", "Orchid", "Dried flowers", "Plant", "Gift basket"]` (no balloons/cakes at a funeral).
3. Score survivors: `+3` if `occasions` includes the chosen occasion; `+1` if discount ≥ 30% (`originalPrice && (1 - amount/originalPrice) >= 0.3`); tiebreak by `id` (localeCompare) for stable output.
4. Diversity cap: max 4 products per category in the result — take by score order, skip a product if its category already has 4, backfill afterwards if fewer than `count` remain.

**`similarTo` logic:**
- With vectors: user vector = element-wise mean of the liked products' embeddings → rank all non-excluded products by cosine similarity → top `count`.
- Fallback (no vectors): score every non-excluded product against the liked set: same category as any liked `+3`, each shared color `+1` (cap 2), price within ±30% of any liked `+1`, shared occasion `+1`; rank desc, tiebreak by id.
- Both paths apply the same sympathy gate as above (pass the occasion in, or expose a `sympathyOk` helper — implementer's choice, but the gate must hold on similar items too).

Create **`apps/web/src/lib/match.test.ts`** (vitest, same style as `recommendations.test.ts`) asserting: (a) `topMatches({occasion:"Birthday"}, new Set())` returns 10 items with zero sympathy products; (b) `topMatches({occasion:"Sympathy"}, ...)` returns only allowed categories; (c) budget filter bounds respected; (d) `similarTo` never returns an excluded id and returns `count` items.
